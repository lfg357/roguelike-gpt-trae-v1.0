# 动画渲染与空间比例规范

本文记录 2026-07-13 动画渲染重构后的正式运行规则。目标不是模仿某个游戏的具体素材，而是达到《暖雪》一类俯视动作游戏所要求的清晰轮廓、快速关键帧、短促能量融合和明确命中反馈。

## 1. 原问题

旧实现每次按 `atlas_size / columns` 得到单元格，再直接绘制整格。但元素、成长反馈和金甲精英图集并不是真正的等宽网格：部分完整姿态跨过了理论格线。等宽裁切会把当前帧右侧切掉，同时采入相邻帧左侧；再叠加人物脚点不一致，视觉上就像下一帧从右侧推走上一帧，并伴有残影和几十像素的中心偏差。

此外，若对整张人物帧做长时间交叉淡化，衣袖、刀身和身体轮廓会同时出现两份，残影反而更重。因此人物与能量特效必须采用不同的过渡策略。

## 2. 时间采样

每条序列统一转换为以下采样结果：

- `frame`：当前关键帧。
- `next_frame`：下一关键帧，循环动作会回到第 0 帧。
- `phase`：当前帧区间内的 0-1 局部时间。
- `progress`：一次性动作的 0-1 总进度。

`idle` 使用 8 fps，`run` 使用 12 fps。攻击、闪避、受击、死亡和特效由动作自身持续时间控制；8 帧普攻在 0.20-0.26 秒内播放，关键峰值仍保持快速、锐利。

命中顿帧期间 `animation_clock` 停止，因此人物、刀光和敌人会共同冻结在命中关键帧；震屏衰减和闪白计时继续更新，不会出现顿帧结束后反馈才突然启动的问题。

## 3. 人物逐帧过渡

人物采用“单帧占优”，任何时刻只绘制一张完整人物贴图：

1. 在 `phase=0.60` 时切换可见人物帧。
2. 可见帧切换前始终使用当前帧锚点，切换后立即使用新帧自己的锚点；禁止在贴图尚未切换时提前插值到下一帧锚点，否则会造成整个人物预滑动。
3. 切换前后不对整个人物做 alpha 叠加，因此不会出现双头、双刀或两套衣袖。
4. `idle` 只附加约 0.8% 的连续呼吸缩放，其他动作约 0.4%，不会用大幅缩放伪造动作。
5. 角色只按 X 轴镜像朝向，不旋转斜俯视人物图。

脚点锚点以源帧归一化坐标记录在 `CharacterSequences.gd`。燕无归和金甲精英使用逐帧锚点轨道；其他角色使用按动作稳定的脚点。普通等宽图集的源锚点会换算到裁边后的区域，金甲精英则使用逐动作真实帧区域和对应锚点。绘制矩形由当前可见帧锚点反推，使碰撞中心、脚下阴影和人物落脚点保持一致。

## 4. 能量特效过渡

刀光、命中、元素和副技能允许极短的相邻帧融合，但融合只发生在当前帧区间末尾：

- `phase < 0.74`：只绘制当前帧，alpha 为 1。
- `phase >= 0.74`：`blend = smoothstep(0.74, 1.0, phase)`。
- 当前帧 alpha 为 `1-blend`，下一帧 alpha 为 `blend`。
- 两帧 alpha 之和恒为 1，不叠亮、不积累尾影。
- 最后一帧不再融合自身，直接保持到序列结束。
- 当前帧与下一帧分别用自己的语义锚点计算目标矩形；即使二者短暂融合，环心、符印中心或命中中心也都固定在同一个局部原点，不会围绕几何中心漂移。

以 0.20 秒、8 帧普攻为例，单帧约 25ms，实际融合窗口约 6.5ms。该窗口只柔化能量轮廓的跳变，不会产生可辨识的双重贴图。

2026-07-14 增补：运行时允许“资源格数”和“有效播放帧数”分离。离线审计发现 `combo_1`、`combo_2`、`combo_3`、`hit`、`crit` 和 `combo_haste` 的第 8 格为透明空尾帧；这些序列仍保留 8 格图集，但播放采样只使用前 7 个有效帧，避免最后 1/8 时间突然消失造成丢帧感。

拾取、武器获得、格局激活和五行圆满脉冲共用 `progression` 重启槽位。新的成长反馈触发时会先移除同槽位旧实例，再从第 0 帧重新播放；命中、暴击、破盾和死亡等战斗反馈仍可多实例叠加，因为它们代表不同命中事件。

## 5. 图集采样与残影消除

角色与 VFX 图集都执行相同的安全采样原则：

1. 真正等宽的图集由 `round(frame * atlas_size / count)` 分别计算起止像素，不累加浮点格宽。
2. `element_actions.png`、`progression_actions.png` 和金甲精英动作表使用离线透明间隔审计得到的逐行真实帧边界。运行时不再把这些资源强行等分。
3. 常规角色向内收 2-3px，VFX 向内收 3px。安全边只用于阻断纹理过滤越界，不能替代真实帧边界，也不能靠加大 padding 掩盖错误分帧。
4. `draw_texture_rect_region` 显式启用 `clip_uv`，禁止 UV 越界采样。
5. 运行时透明 PNG 与绿幕源文件继续分目录保存；不得直接加载绿幕源图。
6. 新图集必须保留透明格间距，不得让武器、墨迹或光效跨越单元格边界。

## 6. 坐标系、视口与加载时机

- 人物、阴影、碰撞体和世界 VFX 都使用 `Main` 的同一设计坐标系；特效位置不经过 HUD `Control`、容器最小尺寸或局部布局坐标。
- 每帧目标矩形为 `Rect2(-anchor * draw_size, draw_size)`。外层 `draw_set_transform(target + camera_offset, angle, scale)` 只负责世界平移和方向；因此旋转、非等比特效尺寸以及窗口缩放都不会改变锚点与目标原点的重合关系。
- 项目固定 1280x720 设计画布并使用 `window/stretch/aspect="keep"`。不同宽高比只产生等比缩放和留边，不会改变世界逻辑范围或单独拉伸 X/Y。
- 鼠标方向使用 `get_global_mouse_position()` 获取 CanvasItem 全局坐标，不直接使用窗口事件坐标，避免 DPI、留边或嵌入窗口改变输入映射。
- 图集使用 `preload`，尺寸元数据在场景运行前即可用；`atlas_matches_texture()` 会校验实际纹理尺寸。锚点不依赖首帧异步读取、控件布局完成信号或运行时透明像素扫描，因此不存在“资源刚加载时尺寸为零”的定位窗口。
- 拾取反馈绑定玩家完成交互时的位置，而不是已经被移除的掉落物旧坐标；允许距离拾取时也不会在角色旁边偏出一截。

Debug 构建按 `F6`（或数字键 `6`）进入锚点审阅场：青色十字是逻辑目标，拾取、五行圆满、护体特效与角色脚点必须逐帧重合。`F7`/`7` 检查成长 VFX 快速连续触发，`F8`/`8` 检查攻击、受击和能量短融合。

`tools/analyze_sprite_anchors.py` 可审计每帧透明边界并给出脚点候选。结果用于人工校正元数据，不在运行时读取像素，避免加载阶段抖动或额外开销。

`tools/validate_animation_sequences.py` 用于深度资源审计，覆盖空帧、近似重复帧、透明像素触边和角色帧间质心大跳变。该脚本只报告风险，不自动判失败；运行时安全裁剪可处理轻微触边，但金甲精英这类主体与披挂/兵刃分离导致的质心跳变，属于后续重制动作表的优先项。

`tests/animation_deep_test.gd` 会遍历全部 VFX 与角色状态，验证真实帧区域有界且不重叠、纹理尺寸匹配、每帧锚点在旋转/非等比尺寸/多种视口缩放下保持原点一致，并覆盖 idle/run、攻击/受击中断、快速重触发、极短持续时间和成长 VFX 重启槽位。

## 7. 命中反馈节奏

- 普通命中：约 60ms 顿帧，平滑衰减震屏，短促暖白闪边。
- 暴击：约 120ms 顿帧，更高震幅和更长衰减。
- 震屏使用正弦组合和 1.7 次幂衰减，不再每帧随机跳点。
- 闪白只覆盖 `ARENA`，不会污染 HUD、地图或文字可读性。
- 打击墨滴、伤害数字、击退继续作为独立反馈层，不替代主动作序列。

碰撞半径同步缩小约 8%-14%，攻击距离和敌人速度保持不变。这样扩大视野但不拉长接敌时间，近战范围在画面上仍然紧凑。右侧地图单元由 36px 调为 32px，信息栏内容宽度调为 228px，二者都保持在独立右栏内。

## 9. 新资源验收标准

只有当现有动作本身缺帧、姿态语义错误或轮廓质量不足时才补制资源。单纯采样、锚点和节奏问题应先在渲染层解决。新增动作表必须满足：

- 同一动作保持稳定脚点和一致透视。
- 每格四周留出透明安全边，任何发光或墨迹不跨格。
- 人物本体不自带长残影；残影由独立语义明确的特效承担。
- 关键动作包含起势、加速、命中峰值、回收，不能只是若干无关姿势。
- 透明化后检查边缘色污染、半透明底色和相邻格泄漏。
