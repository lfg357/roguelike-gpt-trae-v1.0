## 1. 回归测试触发条件

以下情况必须执行完整回归测试：

- [ ] P0 / P1 / P2 里程碑发布前
- [ ] Main.gd 或核心模块（Player/Enemy/Room/Relic/FiveElementDisk/CombatFeedback）发生结构性修改
- [ ] 五行规则系统（FiveElementRules.gd）修改
- [ ] 敌人AI或数值平衡调整
- [ ] 层间切换逻辑修改
- [ ] 遗物系统数据结构变更
- [ ] 动画/VFX系统渲染逻辑变更
- [ ] 输入系统或UI交互逻辑变更

## 5. 精简回归策略

当修改范围明确较小时，可采用精简回归：

| 修改内容 | 必测范围 | 可跳过 |
|----------|----------|--------|
| 仅UI文字/位置调整 | R-44~R-50 | R-01~R-43 |
| 仅敌人数值微调 | R-06~R-13, R-30~R-36 | 五行/遗物/动画 |
| 仅新增遗物 | R-21~R-28 | 层间切换/动画 |
| 仅修复特定Bug | 该Bug相关用例 + 冒烟测试 | 其他无关联用例 |
