# 2. 项目背景（Background）

《仙途·墨渊》是一款修仙背景的俯视角 Roguelike 游戏，使用 Godot 4.7 引擎开发。项目目标是将设计文档 v4.0 中的 P0 游戏闭环逐项做成可玩的 Godot 原型，涵盖探索、战斗、五行遗物构筑、武器抉择、商店与层间结算等核心系统。

项目当前处于 P0 可玩骨架与附录 G 美术基础落地阶段。早期美术实现以静态图集叠加为主，未严格满足附录 G 的逐帧动作要求；2026-07-13 起启动动画渲染重构，将角色和特效改为独立动作序列与时间特效序列，并建立了完整的动画渲染规范与验收体系。

项目不使用 Git 版本控制，所有资源和修改均在本地目录中维护，因此对资源操作持谨慎态度，不删除用户提供的原始资源。

---

# 3. 项目愿景（Vision）

打造一款具有统一深色水墨美学、扎实战斗手感与深度五行构筑系统的俯视角 Roguelike 游戏。玩家在墨渊秘境中探索、战斗、收集遗物与武器，通过五行相生相克与标签组合形成独一无二的构筑，在层层递进的挑战中体验修仙题材的独特韵味。

---

# 4. 项目目标（Objectives）

**Objective 1**：完成 P0 可玩骨架
完成标准：主菜单→开局→探索→战斗→Boss→结算→死亡/胜利的完整闭环可运行，冒烟测试通过。

**Objective 2**：落地附录 G 美术基础
完成标准：角色逐帧动作、核心特效序列、副技能专属序列、成长反馈序列全部接入运行时，通过附录 G 验收表与实机视觉检查。

**Objective 3**：建立可维护的项目架构
完成标准：Main.gd 拆分为 Player/Enemy/Room/Relic/FiveElementDisk/CombatFeedback 等独立模块，文档与代码保持一致，新 Agent 可在 30 分钟内恢复项目状态。

# 5. 项目范围（Scope）

## 包含内容（In Scope）
- P0 核心玩法闭环（探索、战斗、构筑、经济、结算）
- 燕无归主角与 5 把正式武器、15 个副技能
- P0 敌人（刀灵、铜甲、镖手、金甲精英、金甲将军 Boss）
- 五行系统（单元素、双元素、三元素、四元素、五行圆满）
- 14 件 P0 通用遗物与标签组合验证
- 水墨 UI 主题与 1280x720 布局
- 附录 G 角色逐帧动作与 VFX 时间序列
- 埋点骨架与测试框架（smoke_test、animation_deep_test）

## 不包含内容（Out of Scope）
- 多人联机
- 完整音效与音乐（当前仅占位）
- 角色高精度立绘与半身像（P0 之后）
- 完整 TileMap 与场景装饰（P0 之后）
- 移动端适配
- 存档与云同步

# 8. 约束条件（Constraints）

**技术约束：**
- Godot 4.7 stable
- GDScript 编程语言
- 1280x720 设计分辨率，stretch mode 为 keep
- 运行时资源使用透明 PNG，绿幕源文件与运行时资源分目录存放

**业务约束：**
- 严格按照《仙途·墨渊》设计文档 v4.0 推进
- 正式配方、机制和美术语义必须来自设计表/设计文档
- 五行高级格局驱逐低级格局
- 五行圆满是均衡小幅提升，不是全属性尖峰
- 武器掉落必须让玩家抉择，不能自动覆盖
- Boss 死亡后先掉落战利品再触发结算

**资源约束：**
- 人力：用户 + AI
- 平台：PC（Windows）
- 当前目录不是 Git 仓库，操作需谨慎

# 10. 项目架构（High-Level Architecture）

项目采用 Godot 单场景（Main.tscn）+ Node2D 主逻辑（Main.gd）架构，当前所有游戏逻辑集中在 Main.gd 中。资源与配置分为以下模块：

- **角色美术**：`scripts/art/CharacterSequences.gd` 管理角色动作序列、图集元数据与脚点锚点
- **特效系统**：`scripts/vfx/VFXSequences.gd` 管理 34 条 VFX 序列、逐帧锚点与融合规则
- **五行规则**：`scripts/FiveElementRules.gd` 管理五行格局判定与效果计算
- **UI 主题**：`scripts/ui/InkTheme.gd` 管理水墨 UI 主题
- **运行埋点**：`scripts/RunTelemetry.gd` 管理游戏数据埋点
- **测试**：`tests/smoke_test.gd` 冒烟测试、`tests/animation_deep_test.gd` 动画深度测试
- **工具**：`tools/` 下的 Python 脚本用于离线资源审计

后续规划将 Main.gd 拆分为 Player、Enemy、Room、Relic、FiveElementDisk、CombatFeedback 等独立模块。
