### 文档定位
本文档是《仙途·墨渊》项目的**唯一设计基准**，可用于从零开始构建完整游戏。

### 分层深度
- 第一/二部分（核心系统）：规格级（可直接实施）
- 第三部分（meta 愿景）：方向级（P2 后细化）
- 第四/五/六/七/八/九部分（新增）：规格级

### 开发方法论

本文档采用**规格驱动 + 分层验证**的开发方法论：

1. **规格优先**：每个系统先明确规格（数值、行为、交互），再进入编码实现
2. **分层推进**：严格按照 P0→P1→P2 顺序，每层通过 playtest 验证后才进入下一层
3. **可验证性**：所有设计决策都附验证标准（可量化 / 可观察 / 可 playtest）
4. **闸口规则**：每层结束时必须通过对应闸口验证，失败则回退重做该层
5. **从零可构建**：任何开发者拿到本文档即可按附录 F 的实施路线图，从零搭建出完整游戏

### 使用方式

| 读者类型       | 建议阅读顺序                                                       | 关注点                       |
| -------------- | ------------------------------------------------------------------ | ---------------------------- |

| P0 阶段开发者  | §2 核心循环 → §4-§7 战斗/五行核心 → 第五部分 Boss → 附录 F P0 任务 | 打击感原型怎么搭             |

| P2/meta 阶段   | 第三部分 §16-§19 → 附录 F P2 任务                                  | 长期养成深度                 |
| 美术/UI 设计师 | §1 设计支柱 → 各系统 UI 规格 → 附录 B 敌人规格 → 第四/五/七/八部分 | 视觉风格与界面布局           |
| QA/测试        | 第六部分 Playtest 协议 → 附录 F 验收标准                           | 怎么测、测什么               |

# 第一部分：设计基础

## §1 设计支柱

四大支柱是项目的仲裁标尺，任何系统设计冲突都回到这四条来权衡。

### 1.1 四大支柱

| 支柱                      | 内涵                                 | 落地锚点                                                           |
| ------------------------- | ------------------------------------ | ------------------------------------------------------------------ |
| **一·落墨即杀**           | 打击感是生命线。单次命中必须有"重量" | 顿帧 / 震屏 / 水墨粒子 / 三段连击 / 瞬移闪避无敌帧                 |
| **二·五行随心**           | 五行反应是构筑核心，"我安排的局"     | 格局匹配（数值底座）+ 标签组合（机制特效）+ 传说配方（惊喜）       |
| **三·每一局都是独特修行** | Roguelike 随机构筑，每局 build 不同  | 五行格局发现 + 标签组合 + 角色多武器多副技能 + 遗物随机 + 隐含配方 |
| **四·留白之境**           | 信息清晰不堆砌，水墨留白哲学         | 探索式地图 + UI 化解认知负荷 + 发现式机制 + 内容量宁缺毋滥         |

### 1.2 仲裁规则（冲突时按此取舍）

- **打击感 vs 系统复杂度** → 打击感优先（若某机制损害打击感，简化机制）
- **五行深度 vs 留白** → 五行深度优先，但必须用 UI 化解认知负荷（不靠阉割机制）
- **随机性 vs 公平性** → 随机性优先，但保证 build 可行性（不掉死局）
- **内容量 vs 留白** → 留白优先（宁缺毋滥，每件内容必须有存在理由）

### 1.3 支柱二的核心设计（v4.0 重构）

**三层叠加方案**是五行系统的核心架构：

| 层级         | 触发依据                      | 效果类型                        | 发现方式           | 深度       |
| ------------ | ----------------------------- | ------------------------------- | ------------------ | ---------- |
| **五行格局** | 五行盘内元素的整体分布        | 元素附带效果为主 + 少量基础属性 | 首次凑齐时发现     | 入门到中阶 |
| **标签组合** | 五行盘内遗物的标签匹配        | 机制特效（冰冻/反弹/召唤等）    | 首次匹配时发现     | 中阶到高阶 |
| **传说配方** | 五行盘内放入特定的 2-3 件遗物 | 强力改变玩法                    | 完全隐含，偶然发现 | 顶层惊喜   |

三层天然难度分层——新手凑元素就有格局效果，中阶研究标签搭配，老手追求传说配方。

## §2 核心循环

### 2.1 三层循环

| 层级          | 时长     | 内容                                        | 验证标准                                                 |
| ------------- | -------- | ------------------------------------------- | -------------------------------------------------------- |
| **Moment**    | 0–30s    | 命中 → 顿帧 → 水墨 → 震屏 → 连击            | 顿帧时长 / 震屏强度 / 连击窗口可量化；玩家访谈"有重量"   |
| **Session**   | 45–70min | 清一层 5×6 迷宫 → Boss → 进层结算 → 5层通关 | 单局时长分布、清房率（目标 60–70% 不全清）、死亡节点分布 |
| **Long-term** | 跨局     | 业力 meta 永久解锁                          | 单角色 build 多样性、业力获取速率、回流率                |

### 2.2 单局时长收敛设计

5×6 网格每层 8–12 战斗房 × 战斗时长/房 = 每层 5–13 分钟。5 层 = 25–65 分钟。

**收敛手段**：
1. **分支取舍**：迷宫连通性规则保证主线可达，支路可选可不选
2. **层间推深压力**：HP 跨层不回满（见 §15），逼玩家放弃支路保命
3. **单房时长上限**：战斗 20–60s，超时强制清场或惩罚
4. **房间清完即锁**：不可回退，逼取舍
5. **非全清策略**：玩家在 HP 和灵石压力下选择性跳过部分房间

### 2.3 Session Loop 闭环

```text
进入备战房间 → 选角色 → 出发
     ↓
进入层 → START 房(环境叙事) → 探索迷宫(战斗/商店/祭坛) → 击败层 Boss → 层间结算屏 → 下一层
     ↓                                                ↑
通关/死亡结算 ←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←
                                              HP 部分保留
                                              灵石结转(可转化为业力)
                                              遗物保留
                                              业力累计
                                              祭坛 buff 清零
```

### 2.4 从零实施指南（核心循环）

| 步骤   | 内容                              | 交付物                            | 验证方式                               |

## §5 战斗系统

### 5.1 打击感量化规格

| 反馈类型   | 普攻  | 暴击             | 处决     |
| ---------- | ----- | ---------------- | -------- |
| 顿帧时长   | 60ms  | 120ms            | 200ms    |
| 震屏强度   | 3.0   | 8.0              | 5.0      |
| 震屏时长   | 150ms | 200ms            | 300ms    |
| 水墨粒子数 | 8–10  | 12–15            | 20+      |
| 伤害数字   | 普通  | 放大 1.5× + 描边 | 全屏墨溅 |

> **实施要点**：顿帧必须用**独立于游戏时间的计时器**实现。震屏用相机偏移实现，粒子用水墨飞溅效果。

### 5.2 伤害链规范

统一伤害接口设计：

```text
函数签名：
  take_damage(amount: float, source: 实体引用, is_crit: bool = false) -> void

调用链：
  攻击判定区域命中 → 获取被命中的实体 → 调用其 take_damage() → 触发受击反馈
```

**设计要求**：
- 所有可受伤实体（玩家 / 敌人 / 可破坏物）都实现此接口
- 命中判定区域与实体的关联必须清晰
- `source` 参数用于记录伤害来源

### 5.3 状态机设计

使用有限状态机（FSM）管理角色行为：

```text
IDLE → MOVE → ATTACK(1/2/3段) → DODGE → HURT → DEAD
```

**状态切换规则**：
- IDLE ↔ MOVE：输入方向即切换
- MOVE → ATTACK：攻击输入，有连击窗口则进入下一段
- ANY → DODGE：闪避输入，有冷却且非硬直
- ANY → HURT：受到伤害，触发受击硬直 + 无敌帧
- ANY → DEAD：HP 归零

**关键参数**：
- 连击窗口：2.5s
- 三段连击：第三段带突进（持续 8–12 帧）
- 闪避：瞬移 + 无敌帧 0.5s + 冷却 1.5s
- 受击：无敌帧 0.3s（独立于闪避）

### 5.4 攻击判定设计

| 项           | 规格                                 |
| ------------ | ------------------------------------ |
| 攻击类型     | 近战扇形 / 远程弹幕 / 范围领域       |
| 判定方式     | 区域重叠检测                         |
| 命中检测频率 | 每攻击动作检测一次                   |
| 多目标       | 支持同时命中多个敌人                 |
| 命中反馈     | 顿帧 + 震屏 + 粒子 + 伤害数字 + 击退 |

## §6 五行反应系统 ★v4.0 完全重构

### 6.1 三层叠加方案

| 层级         | 触发依据               | 效果类型                        | 效果空间   |
| ------------ | ---------------------- | ------------------------------- | ---------- |
| **五行格局** | 五行盘内元素的整体分布 | 元素附带效果为主 + 基础属性为辅 | 数值底座层 |
| **标签组合** | 盘内遗物的标签匹配     | 机制特效                        | 机制特效层 |
| **传说配方** | 盘内特定遗物组合       | 强力改变玩法                    | 顶层惊喜   |

三层同时生效、互不干扰、互不竞争同一数值空间。

### 6.2 五行格局匹配机制

五行盘 5 个槽位中遗物的元素分布 → 系统自动扫描 → 匹配格局配方 → 生效。

**格局类型分类**：

| 分类       | 数量   | 等阶 | 特点                   |
| ---------- | ------ | ---- | ---------------------- |
| 单元素极致 | 5      | 5    | 最强单一效果，极端赌注 |
| 相生双元   | 5      | 3    | 稳定增益，无负面       |
| 相克双元   | 5      | 3–4  | 高风险高回报，双刃剑   |
| 三元相生链 | 5      | 4    | 联动循环，正反馈链条   |
| 混合三元   | 5      | 4    | 矛盾转化，复杂有趣     |
| 四元格局   | 3      | 5    | 接近圆满但有弱点       |
| 五行圆满   | 1      | 5    | 终极格局，全面强大     |
| **总计**   | **29** |      |                        |

**多格局叠加**：同一组盘内遗物可同时满足多个格局条件，所有满足的格局同时生效。

**发现机制**：首次触发新格局时 → 屏幕边缘提示条淡入（不暂停）→ 之后在图鉴中可查看。

**不给提示**：五行盘 UI 不显示"即将触发"或"还差什么"的提示。格局配方由玩家自行发掘。

| -------- | -------------------------------------- |

| 典型效果 | 解锁终极技能 / 全屏效果 / 召唤特殊单位 |

### 6.5 可扩展配方框架

所有配方（格局/标签组合/传说配方）均采用**数据驱动**设计，新增配方只需追加数据，无需修改核心逻辑。

```text
格局数据结构：
{
    id: 字符串（唯一标识）
    name: 字符串（显示名称）

{
    id: 字符串
    name: 字符串
    required_tags: 字符串数组（需要匹配的标签）
    effect: 字典（生效效果）
    tier: 整数
}

传说配方数据结构：
{
    id: 字符串
    name: 字符串
    required_relics: 字符串数组（需要的遗物ID）
    effect: 字典（生效效果）
    mechanic_change: 字符串（机制改变描述）
}
```

## §7 五行盘系统

### 7.1 设计规格

| 项       | 规格                                   |
| -------- | -------------------------------------- |
| 槽位数   | 5 个（固定不变）                       |
| 槽位性质 | 中性槽位（不绑定元素方位）             |
| 放置规则 | 任意遗物可放入任意槽位                 |
| 遗物标签 | 每件遗物 1–4 个标签（随品级递增）      |
| 遗物元素 | 每件遗物 1 个五行属性                  |
| 效果计算 | 格局 + 标签组合 + 传说配方三层同时生效 |

### 7.2 操作方式

| 平台   | 操作                                                                                                      |
| ------ | --------------------------------------------------------------------------------------------------------- |
| PC     | Tab 打开 → **拖动**背包遗物到目标槽位放置 / **点击**背包遗物再点击槽位放置 / **右键**槽位遗物取出到背包   |
| 移动端 | 左上角按钮打开 → **长按拖动**背包遗物到目标槽位 / **点击**选中再点击槽位放置 / **长按**槽位遗物取出到背包 |

### 7.3 拖动放置规则

- 拖动遗物到已有遗物的槽位：新遗物进入该槽位，旧遗物自动退回背包
- 背包已满时拖动操作被拒绝 + 屏幕提示"背包已满，请先清理"
- 两个槽位互换：将一个槽位的遗物拖到另一个已有遗物的槽位，两件遗物互换位置
- 槽位变化瞬间触发格局/标签组合/传说配方全量重算

### 7.4 拖动过程中的实时预览

| 预览内容     | 表现形式                                          |
| ------------ | ------------------------------------------------- |
| 格局变化     | 拖动过程中五行盘连线颜色实时变化（相生绿/相克红） |
| 标签组合变化 | 如果放下后会触发新标签组合 → 目标槽位边缘闪光     |
| 放置被拒绝   | 遗物弹回原位 + 轻微震动反馈                       |

### 7.5 发现机制

- 首次触发新格局/标签组合/传说配方 → 屏幕边缘提示条淡入（不暂停）
- 已发现的格局/配方可在图鉴中查看
- 未发现的显示"？？？"
- 五行盘 UI 不显示"即将触发"或"还差什么"的提示
- 格局配方由玩家自行发掘

### 7.6 战斗中可操作

- 五行盘**随时可打开**，包括战斗中
- 打开五行盘时游戏**不暂停**
- 移动端五行盘占满全屏，PC 端为半屏弹窗

| 等阶 | 视觉反馈强度                    |

## §16 异化能力系统

### 16.1 定位
build深度层，超越§8的基础build路径。异化能力**改变角色玩法机制**，非纯数值强化。

P2通过业力树解锁异化能力池。

## §17 祖物传承系统

### 17.1 定位
meta永久装备，跨局传承感。

### 17.2 机制
- **封存**：单局达成成就 → 花业力将关键遗物封存为祖物

- **限制**：祖物槽1个，封存有递增业力成本

### 21.5 Boss 5：沉渊之主（第5层·土域）

**教学目标**：终极综合考验 + 全新机制"深渊侵蚀"

**意象**：镇渊石碑本身，操控大地+深渊能量。

**核心新机制——深渊侵蚀**：
- 地面周期性出现深渊裂缝 → 3秒后变为深渊区域
- 站在上面每秒8伤害 + 引力拉扯 + 移速-40%
- 深渊区域缓慢扩大
- 关闭方法：用克制元素攻击边缘

**场地**：矩形大房间，中央石碑（Boss本体不可移动）+ 4根石柱 + 石碑4条裂缝延伸。

**数值**：HP 800 / 元素土 / 护盾200(木克土×2) / 伤害20-35 / 深渊生成频率Phase1:20s/Phase2:12s/Phase3:8s

```text

  C: 深渊召唤 → 生成深渊区域(首个)
  D: 碎石风暴(1.2s) → 石碑周围风暴圈(4秒) → 硬直2秒

  石碑继续A/B/D（频率降低）+ 深渊之影近战追击
  E: 墨爪(0.5s) → 扇形抓击(22伤害)
  F: 深渊之手(0.8s) → 束缚1.5秒+10伤/秒
  G: 渗入地面(3秒不可攻击) → 冲出地面(25伤害)
  深渊召唤每12秒
  30% → Phase 3转换（石碑碎裂，沉渊之主完整体出现，双元素土+水交替）

Phase 3（30%-0% HP）—— "深渊降临"
  元素每15秒切换（切换时2秒虚弱）
  H: 践踏(0.8s) → 3圈地震波
  I: 深渊之握(1.0s) → 4条触手追踪(HP30/条)

  K: 深渊吐息(1.0s) → 120°扇形能量流(2秒) → 摧毁石柱
  15%: 暴怒+深渊区域不可关闭（最终冲刺）
```

### 21.6 Boss 6：心魔镜像（通关后·可选挑战）

**定位**：通关后可选Boss，击败有成就/特殊遗物奖励。

**意象**：复制玩家当前build的镜像。玩家越强镜像越强。

**镜像生成算法**：
1. 复制五行盘内所有遗物（含格局效果）
2. 复制已触发的标签组合/传说配方
3. 复制武器类型+副技能（镜像版外观）
4. HP = 玩家最大HP × 3
5. 伤害 = 玩家伤害 × 1.2
6. 移速 = 玩家移速 × 1.1
7. AI：中距离用投射，近距离用连击，30%概率闪避攻击

**场地**：正方形大房间，中央大型水墨镜屏（不可摧毁主掩体）+ 石柱 + 4块高地。

**战斗脚本**：

```text
Phase 1（100%-60% HP）—— "模仿"
  忠实模仿玩家操作模式
  每20秒留"心魔印记"（踩到攻击力-20%）
  镜屏可被双方攻击打碎(10次) → 碎裂后全屏灵魂能量(双方+50%攻速但每秒10伤)

Phase 2（60%-30% HP）—— "扭曲"
  攻击频率+30%，闪避后留残影(0.5秒后爆炸)
  新增招式：心魔呼唤(召唤1个分身HP50，持续10秒)
  印记频率增加(每12秒)

Phase 3（30%-0% HP）—— "心魔现形"
  全屏冲击波摧毁所有掩体，不再生成新掩体
  暴怒：攻+50%/速+30%/不闪避纯进攻
  元素五行圆满
  纯裸打终局——没有掩体，只有操作
```

**通关后选择**：击败第5层Boss后弹出"返回备战房间"或"挑战心魔镜像"。
